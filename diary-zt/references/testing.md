# 跨设备测试

1. 将整个 `diary-zt` 文件夹复制到目标工具的个人 skill 目录并重启工具。
2. 创建 `<home>/diary-zt/current_path.txt`，内容写入一个**已存在**目录；运行 `/diary-zt 测试`，确认文件创建在该目录。
3. 删除配置文件，重新运行 `/diary-zt 测试`，确认文件创建在 `<home>/diary-zt/yearYYYY/MM/`。
4. 将配置指向不存在目录，确认仍回退默认目录而非创建该配置目录。
5. 在同一会话运行 `/diary-zt` 两次，确认生成两份不同的完整快照。
6. 首次快照后发送一条新消息，运行 `/diary-zt-a`，确认只追加新消息；立即再运行一次，确认报告“没有新增内容”。
7. 使用 `/diary-zt-a 已有文件名.md`，确认仅向该已有文件追加；对不存在文件名确认不会创建文件。
8. 在 Windows、macOS 和 Linux 分别确认结果位于当前用户主目录下的 `diary-zt`。
